# Project Structure

This file explains the purpose of each major folder and file in the repository.

## Root

- `README.md`: Project overview, quick start, API usage, and deployment instructions.
- `requirements.txt`: Python dependencies for local development and runtime.
- `pyproject.toml`: Package metadata and tooling configuration.
- `train.py`: Main training entrypoint for model development.
- `docker-compose.yml`: Multi-service orchestration for API, dashboard, and MLflow.
- `Dockerfile.api`: Container build file for the FastAPI service.
- `Dockerfile.ui`: Container build file for the Streamlit dashboard.
- `.env.example`: Template environment variables.
- `.gitignore`: Git ignore rules.
- `LICENSE`: MIT license.
- `hospital_cost_prediction.ipynb`: Notebook-based analysis/prototyping workflow.

## Application Code

- `src/config.py`: Centralized environment-driven settings.

### Data Layer (`src/data/`)

- `loader.py`: Loads SPARCS dataset and generates synthetic fallback data when the dataset is unavailable.
- `preprocessor.py`: Builds and persists sklearn preprocessing pipeline components.

### Model Layer (`src/models/`)

- `traditional.py`: Traditional ML model training and evaluation (Ridge, Random Forest, XGBoost, LightGBM).
- `neural.py`: PyTorch-based neural network regressor.
- `trainer.py`: Training orchestration, metrics collection, artifact persistence, and MLflow logging.
- `explainer.py`: Model explainability utilities.

### API Layer (`src/api/`)

- `main.py`: FastAPI app startup, model loading, and inference endpoints.
- `schemas.py`: Pydantic request/response schemas for API contracts.

## Frontend

- `app/streamlit_app.py`: Interactive dashboard for single and batch predictions.
- `.streamlit/config.toml`: Streamlit runtime configuration.

## Data and Artifacts

- `data/`: Source dataset location.
- `artifacts/`: Saved trained models, preprocessing pipelines, metrics, and related outputs generated by training.
